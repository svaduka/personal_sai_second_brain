# 💭 Fleeting Notes

Fleeting notes are temporary captures—quick thoughts, meeting notes, and daily logs that may or may not become permanent knowledge.

## Purpose

- **Daily documentation**: Track daily activities and thoughts
- **Quick captures**: Ideas that need time to develop
- **Meeting notes**: Conversations and decisions
- **Temporary thoughts**: Ideas being explored

## Structure

```
01-Fleeting/
└── Daily/
    ├── 2026-04-21.md
    ├── 2026-04-22.md
    └── ...
```

## Daily Notes

### Location
`01-Fleeting/Daily/YYYY-MM-DD.md`

### When to Use
- **Every day**: Create a daily note for today
- **Quick captures**: Jot down thoughts as they come
- **Work log**: Track what you did
- **Reflections**: End-of-day thoughts

### Daily Note Workflow

**Morning:**
1. Create today's daily note (use template)
2. Set daily intention and top priority
3. Review yesterday's note for carryovers

**Throughout the day:**
1. Capture thoughts in quick captures section
2. Log work activities
3. Add tasks as they come up

**Evening:**
1. Review what got done
2. Add reflections
3. Note tomorrow's priority

## Meeting Notes

Meeting notes can go here if they're temporary. If a meeting spawns actionable project work, create a project note in `05-Projects/` instead.

### Meeting Note Format
```markdown
# Meeting: [Topic] - YYYY-MM-DD

**Attendees**: [Names]
**Purpose**: [Why we met]

## Notes
- Key point 1
- Key point 2

## Decisions
- Decision 1
- Decision 2

## Action Items
- [ ] Task 1 - Owner: [Name]
- [ ] Task 2 - Owner: [Name]

## Follow-up
- [[Related-Project]]
- [[Related-Note]]
```

## Fleeting vs. Permanent

**Fleeting notes are:**
- ✅ Quick and rough
- ✅ For yourself today
- ✅ Temporary by nature
- ✅ Unrefined thoughts
- ✅ Time-bound (daily, weekly)

**Permanent notes are:**
- ✅ Refined and clear
- ✅ For your future self
- ✅ Timeless knowledge
- ✅ Atomic ideas
- ✅ Well-connected to other notes

## Processing Fleeting Notes

**Weekly review** of fleeting notes:
1. Read through last week's daily notes
2. Extract valuable insights
3. Transform into permanent notes
4. Link to existing knowledge
5. Archive or delete old fleeting notes

**What to do with fleeting notes:**
- **Valuable insight** → Create permanent note
- **Project-related** → Add to project note
- **Reference material** → Move to resources
- **No longer relevant** → Archive or delete

## Tips

### Daily Note Consistency
- Use the same template every day
- Review yesterday's note each morning
- Keep entries brief—this isn't a journal
- Focus on knowledge, not just activities

### Quick Capture Tips
- Write full sentences (future you will thank you)
- Add context: where did this thought come from?
- Tag with `#process` if it needs follow-up
- Link to related notes immediately if obvious

### Avoid Daily Note Bloat
- Don't try to capture everything
- Focus on notable thoughts and learnings
- Use task management tools for todos (don't duplicate)
- If something is worth keeping, make it a permanent note

## Common Mistakes

❌ **Treating daily notes as permanent**: Daily notes are disposable
- ✅ Extract valuable ideas to permanent notes

❌ **Writing essays in daily notes**: Keep them brief
- ✅ Detailed thinking → separate permanent note

❌ **Never reviewing daily notes**: They won't process themselves
- ✅ Weekly review extracts the gold

❌ **Skipping daily notes entirely**: Missing opportunities to capture
- ✅ Even a 30-second capture is valuable

## Lifespan

- **Daily notes**: Keep for 3-6 months, then archive or delete
- **Meeting notes**: Keep while relevant to active projects
- **Fleeting thoughts**: Process within 1-2 weeks

**Archive old daily notes:**
```

```

**Best Practice**: Think of daily notes as a capture net—catch everything, then sift through weekly to find the gems worth keeping permanently.
