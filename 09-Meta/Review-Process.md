# Review Process

Regular review is essential to maintaining a healthy second brain. This document outlines the daily, weekly, monthly, and quarterly review processes to keep your knowledge system flowing and growing.

## Review Philosophy

**Why review matters:**
- **Inbox stays manageable**: Regular processing prevents overwhelm
- **Connections emerge**: Reviewing notes reveals relationships
- **Knowledge compounds**: Regular engagement strengthens understanding
- **System evolves**: Reviews help you refine what works

**Time investment:**
- Daily: 5 minutes
- Weekly: 30 minutes
- Monthly: 60 minutes
- Quarterly: 2 hours

**Return:** A living, breathing knowledge system that serves you well

## Daily Review

**Duration**: 5 minutes
**Best time**: End of day
**Frequency**: Every weekday (weekends optional)

### Daily Checklist

- [ ] Review today's daily note
- [ ] Capture any remaining thoughts
- [ ] Check active project status (quick glance)
- [ ] Note tomorrow's top priority

### Daily Process

1. **Open today's daily note** (`01-Fleeting/Daily/YYYY-MM-DD.md`)
2. **Scan quick captures**: Any ideas worth expanding?
3. **Review tasks**: What got done? What carries over?
4. **Reflections**: Add a quick end-of-day reflection
5. **Preview tomorrow**: Set intention for tomorrow

### Daily Note Workflow

```markdown
End of day review:
- ✅ Captured 3 new ideas
- ✅ Completed priority task
- 📝 Tomorrow: Focus on [specific task]
```

**Don't:**
- Process inbox notes (save for weekly)
- Create permanent notes (unless inspiration strikes)
- Perfect formatting (keep it quick)

## Weekly Review

**Duration**: 30 minutes
**Best time**: Sunday evening or Friday afternoon
**Frequency**: Once per week (non-negotiable)

### Weekly Checklist

- [ ] Process inbox (target: under 10 items)
- [ ] Transform 2-3 fleeting notes into permanent notes
- [ ] Update active project notes
- [ ] Review last week's daily notes
- [ ] Create/update links between notes
- [ ] Plan next week's focus

### Weekly Process

#### 1. Inbox Processing (15 minutes)

**For each inbox item, decide:**

```
Is this still relevant?
├─ No → Delete or archive
└─ Yes → What type?
    ├─ Quick thought → Make permanent or discard
    ├─ Source material → Move to 02-Literature/
    ├─ Refined idea → Move to 03-Permanent/
    ├─ Project-related → Move to 05-Projects/
    └─ Reference → Move to 07-Resources/
```

**Target**: Get inbox to 10 or fewer items

#### 2. Note Creation (10 minutes)

- Pick 2-3 promising fleeting notes or inbox items
- Transform into permanent notes using template
- **Critical**: Add links to at least 1-2 existing notes
- Update any relevant MOCs

#### 3. Project Updates (5 minutes)

For each active project:
- Add weekly progress log entry
- Update next actions
- Note any blockers
- Adjust timeline if needed

### Weekly Review Template

Copy this to your daily note on review day:

```markdown
## Weekly Review

### Inbox Status
- Started with: [X] items
- Processed: [X] items
- Remaining: [X] items

### Notes Created
- [[New-Permanent-Note-1]]
- [[New-Permanent-Note-2]]

### Projects Updated
- [[Project-1]] - Status: On track
- [[Project-2]] - Status: Blocked by [X]

### Next Week Focus
- Priority 1:
- Priority 2:
- Priority 3:

### Observations
- What's working:
- What needs adjustment:
```

## Monthly Review

**Duration**: 60 minutes
**Best time**: Last day of the month
**Frequency**: Once per month

### Monthly Checklist

- [ ] Review all MOCs for completeness
- [ ] Archive completed projects
- [ ] Check for orphaned notes (no incoming links)
- [ ] Identify emerging topics for new MOCs
- [ ] Update Master Index statistics
- [ ] Review and update areas of responsibility
- [ ] Clean up attachments folder
- [ ] System health check

### Monthly Process

#### 1. MOC Review (20 minutes)

For each MOC in `04-MOCs/`:
- **Read through**: Does it still make sense?
- **Check links**: Are there broken links?
- **Add new notes**: Any permanent notes missing from this MOC?
- **Reorganize**: Should any section be restructured?
- **Split if needed**: MOC over 50 notes? Consider splitting

**Update each MOC:**
```markdown
Last Reviewed: 2026-04-21
Note Count: 23
Status: 🌿 Developing
```

#### 2. Project Archival (10 minutes)

Review `05-Projects/`:
- **Completed projects**: Move to `08-Archive/YYYY/MM/`
- **Inactive projects** (no update in 2+ months): Consider archiving or closing
- **Extract learnings**: Create permanent notes from project insights

**Archival checklist:**
- [ ] Document final outcomes
- [ ] Extract learnings to permanent notes
- [ ] Update status to "Completed" or "Cancelled"
- [ ] Move to archive with date folder

#### 3. Orphan Note Check (10 minutes)

Find notes with no incoming links:
1. In Obsidian: Use Graph View or search `file:""` with backlinks filter
2. For each orphan: Is it valuable?
   - Yes → Create links from relevant notes or MOCs
   - No → Archive or delete

#### 4. Emerging Topics (10 minutes)

Scan `03-Permanent/`:
- Notice clusters of related notes?
- Do you have 10+ notes on a new topic?
- **If yes**: Create a new MOC

**Signs you need a new MOC:**
- You're repeatedly searching for related notes
- A topic keeps coming up
- You have 10+ notes on a theme
- Navigation is getting difficult

#### 5. Master Index Update (5 minutes)

Update `04-MOCs/00-Master-Index.md`:
- **Total Permanent Notes**: Count notes in `03-Permanent/`
- **Total MOCs**: Count notes in `04-MOCs/`
- **Active Projects**: Count active projects
- **Unprocessed Inbox Items**: Count items in `00-Inbox/`
- **Add new MOCs** to navigation sections

#### 6. System Health (5 minutes)

Check these indicators:

| Metric | Healthy | Warning | Action Needed |
|--------|---------|---------|---------------|
| Inbox items | < 10 | 10-20 | > 20 |
| Days since last permanent note | < 7 | 7-14 | > 14 |
| Orphaned notes | < 5% | 5-10% | > 10% |
| Active projects | 1-5 | 6-8 | > 8 |
| MOCs without recent review | 0 | 1-2 | > 2 |

## Quarterly Review

**Duration**: 2 hours
**Best time**: First day of new quarter
**Frequency**: Every 3 months (Jan, Apr, Jul, Oct)

### Quarterly Checklist

- [ ] Review system conventions
- [ ] Assess what's working and what isn't
- [ ] Clean up archive folder
- [ ] Refactor growing MOCs
- [ ] Update system documentation
- [ ] Backup entire system
- [ ] Review permanent notes growth
- [ ] Plan next quarter's focus areas

### Quarterly Process

#### 1. System Assessment (30 minutes)

**Reflect on the past quarter:**

**What's working well?**
- Which workflows are smooth?
- What habits have stuck?
- Which conventions are helpful?

**What's not working?**
- What feels friction-filled?
- Which rules do you keep breaking?
- What's confusing or unclear?

**Adjustments to make:**
- Update conventions if needed
- Simplify complex workflows
- Remove unused templates or folders

**Document in**: `09-Meta/Quarterly-Review-YYYY-QX.md`

#### 2. Convention Review (20 minutes)

Review `09-Meta/Conventions.md`:
- Are naming conventions being followed?
- Do linking strategies make sense?
- Should any rules be changed?
- Are tags being used appropriately?

**Update conventions** if needed, with rationale:
```markdown
## Change Log

### 2026-Q2
- **Changed**: Permanent note naming from [old] to [new]
- **Reason**: [Why this change helps]
- **Impact**: [What needs to be updated]
```

#### 3. MOC Refactoring (30 minutes)

For large MOCs (30+ notes):
- **Consider splitting**: Break into topic-specific MOCs
- **Reorganize**: Better structure for easier navigation
- **Create sub-MOCs**: Hierarchical organization if needed

**Example refactoring:**
```
Programming-MOC (50 notes)
├─ Split into:
│   ├─ Programming-Languages-MOC (20 notes)
│   ├─ Programming-Paradigms-MOC (15 notes)
│   └─ Programming-Tools-MOC (15 notes)
```

#### 4. Archive Cleanup (15 minutes)

Review `08-Archive/`:
- Organize by year/month if not already
- Delete truly obsolete items
- Consider permanently removing old drafts
- Keep completed projects for reference

#### 5. Growth Analysis (15 minutes)

**Track your knowledge base growth:**

| Metric | This Quarter | Last Quarter | Growth |
|--------|--------------|--------------|--------|
| Permanent Notes | | | |
| MOCs | | | |
| Projects Completed | | | |
| Average Weekly Reviews | | | |

**Insights:**
- Are you creating permanent notes regularly?
- Is your knowledge base growing intentionally?
- Are you making connections between ideas?

#### 6. Next Quarter Planning (10 minutes)

**Set intentions for next quarter:**
- Focus areas or topics to explore
- System improvements to implement
- Habits to develop
- Projects to start or complete

**Document in**: `09-Meta/Quarterly-Plan-YYYY-QX.md`

## Review Best Practices

### Make it Sustainable

1. **Schedule it**: Add reviews to calendar with reminders
2. **Time-box**: Use a timer, don't let reviews expand
3. **Start small**: Miss a review? Do a shorter version, don't skip
4. **Forgive gaps**: Life happens—just resume when you can

### Make it Effective

1. **Focus on value**: Process notes that matter, delete the rest
2. **Create connections**: Every review should add links
3. **Reflect honestly**: If something isn't working, change it
4. **Celebrate growth**: Notice how much you've learned

### Make it Habitual

1. **Same time, same place**: Build a routine
2. **Trigger-action**: After [event], I do [review]
3. **Track the habit**: Mark off reviews in daily notes
4. **Adjust as needed**: Find what works for you

## Troubleshooting Reviews

### "I don't have time for reviews"

- Start smaller: 5 min daily, 15 min weekly
- Combine with existing habits (coffee time, commute)
- Remember: Reviews save time by preventing chaos

### "My reviews are too long"

- Time-box strictly: Use a timer
- Focus on essentials only
- Batch similar tasks
- Skip optional sections when busy

### "I keep skipping reviews"

- Make it easier: Reduce scope
- Make it rewarding: Treat yourself after
- Make it visible: Calendar reminders
- Make it accountable: Share progress with someone

### "Reviews feel like busywork"

- Focus on value: Process, don't just organize
- Create connections: Make reviews generative
- Reflect honestly: Change what doesn't work
- Celebrate wins: Notice your growth

---

**Remember**: The goal of reviews isn't perfection—it's a living, useful knowledge system. Start with what you can sustain, adjust as you go, and let the system serve you.
