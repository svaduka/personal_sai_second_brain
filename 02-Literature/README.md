# 📚 Literature Notes

Literature notes capture ideas from external sources—books, articles, courses, videos, and podcasts. They bridge the gap between someone else's ideas and your own understanding.

## Purpose

- **Capture source material**: Notes from what you read, watch, or listen to
- **Record key takeaways**: Main ideas from external sources
- **Enable processing**: Transform source ideas into your own permanent notes
- **Maintain references**: Track where ideas originated

## When to Use

✅ **Create literature notes when:**
- Reading a book that contains valuable ideas
- Reading an insightful article or blog post
- Taking a course (online or in-person)
- Watching educational videos
- Listening to podcasts with noteworthy content
- Reading academic papers or research

❌ **Don't create literature notes for:**
- Fiction books (unless analyzing themes/techniques)
- Entertainment content
- Social media posts (capture directly to inbox if valuable)
- Quick tips (add directly to resources)

## Literature Note Workflow

### 1. During Consumption

**As you read/watch/listen:**
- Take rough notes (can be in inbox first)
- Highlight key passages
- Note page numbers or timestamps
- Mark ideas that resonate
- Write questions that arise

### 2. After Consumption

**Create structured literature note:**
1. Use the literature note template
2. Summarize in your own words
3. Extract key takeaways (3-7 main points)
4. Add notable quotes with references
5. Write personal reflections
6. Note disagreements or questions

### 3. Processing Phase

**Transform into permanent notes:**
1. Identify atomic ideas worth developing
2. Create separate permanent notes for each idea
3. Write each idea in your own words
4. Link permanent notes to literature note
5. Link permanent notes to related concepts

### 4. Maintenance

**Track what you've processed:**
- Update "Permanent Notes Created" section
- Mark status as "Completed" when fully processed
- Link to any projects or MOCs updated

## Naming Conventions

### Books
**Format**: `Author-Year-Book-Title.md`
- Example: `Newport-2016-Deep-Work.md`
- Example: `Clear-2018-Atomic-Habits.md`

### Articles
**Format**: `Source-YYYY-MM-DD-Article-Title.md`
- Example: `Medium-2026-04-15-Building-Second-Brain.md`
- Example: `NYT-2026-03-10-Productivity-Myths.md`

### Courses
**Format**: `Platform-Instructor-Course-Name.md`
- Example: `Coursera-Andrew-Ng-Machine-Learning.md`
- Example: `Udemy-Bob-Ross-Joy-Of-Painting.md`

### Videos/Podcasts
**Format**: `Platform-Creator-YYYY-MM-DD-Title.md`
- Example: `YouTube-Ali-Abdaal-2026-04-10-Note-Taking.md`
- Example: `Podcast-Tim-Ferriss-2026-04-05-Derek-Sivers.md`

### Academic Papers
**Format**: `FirstAuthor-Year-Paper-Title.md`
- Example: `Kahneman-2011-Thinking-Fast-And-Slow.md`

## Literature to Permanent Note Process

**Literature note** (source's words):
```markdown
# Newport-2016-Deep-Work

## Key Takeaway
Deep work is the ability to focus without distraction on
cognitively demanding tasks.
```

**Permanent note** (your words):
```markdown
# 20260421150000-deep-work-creates-value

Deep work—sustained, focused effort—is increasingly rare and
increasingly valuable. In a distracted world, the ability to
concentrate deeply is a competitive advantage.

Related to: [[attention-as-resource]], [[flow-state]]
```

## Quality Guidelines

### Good Literature Notes

✅ **Summarize, don't copy**: Write in your own words
✅ **Capture context**: Why you read this, what you were looking for
✅ **Include metadata**: Author, date, source URL
✅ **Add reflections**: Your thoughts, agreements, disagreements
✅ **Link forward**: Connect to permanent notes created from it
✅ **Quote sparingly**: Only the most impactful passages

### Avoid These Pitfalls

❌ **Highlighting without processing**: Highlights aren't understanding
❌ **Copying verbatim**: Doesn't help you internalize
❌ **No personal reflection**: What do YOU think?
❌ **Processing immediately**: Let ideas marinate first
❌ **Trying to capture everything**: Focus on what's relevant to you

## Types of Literature Notes

### Reference Style
Short, focused on specific information:
- API documentation
- Technical specifications
- Quick reference guides
- Statistical data

### Learning Style
Comprehensive, focused on understanding:
- Textbooks
- In-depth courses
- Complex concepts
- Step-by-step tutorials

### Insight Style
Idea-focused, focused on concepts:
- Books with big ideas
- Thought-leadership articles
- Podcasts with insights
- Research papers

## Progressive Summarization

Layer your notes in stages:

**Stage 1**: Rough highlights while consuming
**Stage 2**: Structured literature note after finishing
**Stage 3**: Extract key takeaways (bold the most important)
**Stage 4**: Create permanent notes from core ideas
**Stage 5**: Link permanent notes into your knowledge system

Don't try to do all stages at once—let it develop over time.

## Organization Tips

### By Source Type
Organize into subfolders:
- Books for long-form reading
- Articles for web content
- Papers for academic sources
- Courses for structured learning
- Videos-Podcasts for multimedia

### By Status
Use status tags or frontmatter:
- `status: Reading` - Currently consuming
- `status: Completed` - Finished, needs processing
- `status: Processed` - Permanent notes created
- `status: Reference` - Keep for lookup, no processing needed

## Common Workflows

### Book Reading Workflow
1. **While reading**: Take rough notes in notebook/kindle
2. **After chapter**: Transfer notes to literature note
3. **After finishing**: Write summary and reflections
4. **Week later**: Create permanent notes from key ideas
5. **Link**: Connect permanent notes to existing knowledge

### Article Reading Workflow
1. **Quick read**: Is this worth noting?
2. **Deep read**: Take structured notes
3. **Same day**: Create literature note with template
4. **Next weekly review**: Create permanent notes if valuable

### Course Workflow
1. **Per lesson**: Add notes to literature note
2. **Per section**: Summarize key learnings
3. **After completion**: Comprehensive summary
4. **Create**: Project note if building something
5. **Extract**: Permanent notes for concepts learned

## Linking Strategy

**From literature notes:**
- Link to permanent notes created: `[[YYYYMMDD-note]]`
- Link to related literature: `[[Other-Author-Book]]`
- Link to relevant MOCs: `[[Topic-MOC]]`

**To literature notes:**
- Permanent notes should cite: `Source: [[Author-Year-Book]]`
- MOCs can include "Key Resources" section
- Project notes reference relevant sources

## Review Process

**During weekly review:**
- Process 1-2 literature notes into permanent notes
- Update status of literature notes
- Archive fully-processed sources

**During monthly review:**
- Check for unprocessed literature notes older than 1 month
- Either process or move to archive if no longer relevant

---

**Remember**: Literature notes are scaffolding, not the building. The goal is to transform others' ideas into your own permanent notes. Don't let literature notes pile up unprocessed—extract the value and move forward.
